# Q-BU スマホ版 要件定義書（v1）

> このドキュメントの目的
> - v1（最初のワークショップ）の **機能要件を確定** する
> - 購入すべき **ハードウェア（ミニPC等）の見積もり** に使えるようにする

---

## 1. 目的・背景

- 子ども向けワークショップ用ツール。
- 子どもが **各自のスマホ** で立方体（ブロック）を並べて3Dモデルを作る。
- 「送信」すると、**ホストPC** で3Dプリント用データ（STL）に変換し、印刷して子どもに渡す。
- 「自分で作ったものが実物になる」体験で喜んでもらうのがゴール。

---

## 2. 利用シーン・前提条件

| 項目 | 内容 |
|---|---|
| 同時利用 | 最大 **約60人** |
| 端末 | 子どもの各自スマホ ＋ 貸出機 5〜6台 |
| ターゲット端末 | **iPhone 11世代（4年落ち）以上** を想定。軽い編集なので余裕 |
| ネットワーク | 会場の **ローカルWi-Fi**。インターネット接続には依存しない |
| プリンター | **単色（1色）・5台並列**（v1時点） |

---

## 3. 全体構成（アーキテクチャ）

```
 ┌─────────────────────┐         ┌──────────────────────────┐
 │  子どものスマホ ×60     │         │     ホスト（ミニPC）          │
 │                      │  ①QRで  │                           │
 │  ・Webアプリを開く      │  URL配布 │  ① Webアプリ配信（静的）      │
 │  ・合言葉を入れる        │ ──────> │  ② 送信受け口（小さなAPI）     │
 │  ・ブロックを並べる      │         │  ③ JSON → STL 変換          │
 │   (全部ローカル＝軽い)   │  ②送信時 │  ④ 印刷キュー / 管理（別構築）  │
 │  ・送信                │  JSONを │  ⑤ 3Dプリンタへ出力          │
 │  ・完了画面            │  POST   │                            │
 └─────────────────────┘ (数KB)   └──────────────────────────┘
        ローカルWi-Fi（インターネット不要）
```

### 設計方針
- **編集はローカル（軽い・ラグなし）／重い処理はホスト集約（STL生成・印刷）**
  → 「端末が重い」も「サーバーがラグい」も両方回避できる。
- 通信は **送信ボタンを押したときだけ**。数KBのJSONをPOSTするだけ。
- データは **バージョン付きJSON**。将来の拡張（複数色・認証）を後から載せられる土台にする。

---

## 4. 利用フロー（子ども側）

1. **QRコード** を読む → アプリのURLが開く
2. **合言葉（パスコード）** を入力（管理者が口頭で伝える）
3. 編集画面が表示 → **タップで直感的に** ブロックを置く／消す
4. **プレビュー**（少し影のつく表示）で確認
5. **名前 or 番号** を入れて「**送信**」
6. 「**完了！**」画面 → 終わり

---

## 5. 機能要件

### 5.1 アクセス／簡易認証
- QRコードでアプリを開く。
- **合言葉（パスコード）ゲート**：常時公開にはしない。**1日単位で固定**（管理者が日替わりで変更）。
- **識別ID**：アクセス時に **自動付与**（短い番号/コード）。端末の localStorage に保存し、**再接続・リロードでも消えない**。名前入力は任意で併用可。
- ※本格的な認証は将来拡張。v1は合言葉＋自動IDのみ（**最小要件**）。

### 5.2 エディタ（コア）
- ブロックの **配置／削除**（タップ操作）。
- **視点回転**（上下左右）、**ズーム**（＋／−）。
- **単色（v1）**。色はデータ上は属性として持ち、将来の複数色に備える。
- 編集可能空間：**最大 100×100×100**（座標範囲。実際は全部は使わない＝データはスパース保持）。
- エッジ表示のオン/オフ（任意）、リセット。

### 5.3 プレビュー
- **少し影（陰影）のつくプレビュー**表示（好評のため継続）。

### 5.4 送信
- 送信データに **自動付与ID**（＋任意で名前）を含める。
- 「送信」でモデルデータ（JSON）をホストへPOST。
- 送信後は「**完了**」画面 →「**あたらしくつくる**」でID・作品をリセット（**貸出機を次の子が使える**ように）。
- 制作中の作品は localStorage に自動保存し、**途中で接続が切れても復元**できる（作品が消えて *かわいそう* を防ぐ）。

### 5.5 ホスト：受信・変換・印刷
- JSON受信・保存。
- JSON（blocks）→ **STL生成**（1ブロック＝1立方体、面の結合で清書）。
- **印刷可能性チェック**：浮いている／分離したブロックの警告（オペレータ確認用）。
- スライサー → 3Dプリンタで出力。

### 5.6 ホスト管理ツール（★別途構築）
- 受信一覧（誰の・いつ）。
- ステータス：受信 → 変換 → 印刷中 → 完了。
- 合言葉（パスコード）の変更。
- ※本体アプリとは **別コンポーネント** として構築。

---

## 6. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | 編集はローカルで軽量、通信は送信時の数KBのみ → **60人同時でも余裕・ラグなし** |
| 可用性 | ローカルネットワークで完結（インターネット不要） |
| 端末互換 | iPhone 11世代〜、主要モバイルブラウザ（Safari / Chrome） |
| データ | スパースJSON、`version` フィールドで将来拡張に対応 |
| セキュリティ | 合言葉ゲートで常時公開を避ける（v1） |

---

## 7. データ仕様（送信JSON）

```json
{
  "version": 1,
  "id": "A-0427",            // アクセス時に自動付与（localStorageで保持）
  "name": "たろう",          // 任意
  "projectName": "",         // 任意
  "createdAt": "<端末の時刻>",
  "color": "#9AA0A6",        // v1は単色
  "blocks": [
    { "x": 0, "y": 0, "z": 0 },
    { "x": 1, "y": 0, "z": 0 }
  ]
}
```

- 座標は整数 **0〜99**（100³空間）。
- `blocks` は「置いてあるブロックだけ」の **スパース配列**（全マスは持たない）。
- 将来：`blocks` の各要素に `color` を持たせれば複数色に対応できる。

---

## 8. v1スコープ外（将来拡張）

- 複数色（Painting）／ Coding ／ Gallery モード
- アカウント登録・公開・保護者同意フロー
- 本格認証（合言葉 → ちゃんとした認証へ）
- 管理ツールの高度化

> 土台（データ形式・構成）は、これらを **後から載せられる作り** にしておく。

---

## 9. 推奨ハードウェア（購入見積もり用）

### ホスト（ミニPC）
- 役割：Web配信 ＋ 受信API ＋ STL生成 ＋ スライス ＋ 管理 ＋ 印刷制御。
- 一番重いのは **スライス処理**（1モデルずつ・CPU依存）。**GPUは不要**。
- 推奨スペック：
  - CPU：最新の **6コア以上**（Apple Mシリーズ / Intel Core i5〜i7 12世代以降 / Ryzen 5〜7）
  - メモリ：**16GB**（8GBでも可、16GB推奨）
  - ストレージ：**256〜512GB SSD**
  - 有線LAN（安定）、USB（プリンタ接続）
- 具体例：
  - **Mac mini（M4 / 16GB）** ← macOS環境ならおすすめ・静音・楽
  - Windows/Linux ミニPC（Core i5/i7 or Ryzen 5/7、16GB、512GB SSD）

### ネットワーク
- 60台前後つなぐので、**安定したルーター/AP**（業務用AP＝UniFi等だと安心）。
- 通信量は軽いが「台数」が多いので、家庭用ルーターより業務用APが無難。

### 印刷スループット（運用で解決済み）
- **プリンタ5台を並列**運用してスループットを確保。
- 刷り切れない分は **後日郵送でOK**。継続的に会う子どもたちなので渡し損ねもカバー可能。
- → 「当日全員分」にはこだわらない運用で確定。

---

## 10. 確定事項

1. **子どもの識別**：アクセス時に **ID自動付与**（localStorage保持＝再接続でも消えない）。名前は任意。
2. **合言葉**：**1日単位で固定**（日替わりで変更）。
3. **印刷物の渡し方**：**5台並列 ＋ 後日郵送OK**。当日全員分にはこだわらない。
