## 1. このプロジェクトは何をしている？

**q-bu.com のスマホ版を新規開発**している。

q-bu は子ども向けワークショップ用ツール。子どもが立方体（ブロック）を並べて3Dモデルを作り、
それを **3Dプリント** して実物を渡して喜んでもらう。

- **v1のゴール**：子どもが各自スマホで「ブロックを並べる → 送信 → 完了」。
  送信データはホストPCに集約 → STL変換 → 3D印刷 → 子どもに渡す。
- 詳細仕様 → [要件定義.md](要件定義.md)
- 将来拡張の方針 → [拡張性設計.md](拡張性設計.md)

## 3. これまでの作業ログ

1. **元サイト調査** … q-bu.com の保存ファイルを解析 → **Next.js + Three.js(r162)** と判明。
   4モード（Modeling / Painting / Coding / Gallery）、印刷準備画面、アカウント・同意フロー等を確認。
2. **要件定義** … [要件定義.md](要件定義.md) で v1要件をフィックス（最大60人・各自スマホ・
   ローカルWi-Fi・単色プリンタ5台並列・JSON→STL→印刷）。
3. **操作モック** … [mock.html](mock.html) を Three.js で実装。タッチ操作を検証
   （面タップ＝置く／つくる・けすトグル／1本指ドラッグ＝回す／ピンチ＝ズーム／視点ボタン）。
4. **送信の仕組み** … [host.py](host.py)。アプリ配信＋送信受け口を**依存ゼロ**で。
   送信ボタンが `/submit` にJSONをPOST → `submissions/` に1人1ファイル保存。
5. **拡張性設計** … [拡張性設計.md](拡張性設計.md)。B2B拡張の土台（seam）を仕込み
   （前方互換スキーマ / CONFIG機能フラグ / ボクセル→メッシュ共有パイプライン）。
6. **git** … ローカルで初期化＆コミット済み。リモートpushは保留（下記）。

## 6. 動かし方

```bash
python3 host.py
# → スマホで  http://<MacのLAN IP>:8000/mock.html  を開く（同じWi-Fi）
```
- 送信データは `submissions/` に JSON で溜まる（1人1ファイル）。
- モックは Three.js を CDN から読むので、試すにはネット接続が必要（本番はローカル同梱予定）。
