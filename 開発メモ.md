## 1. このプロジェクトは何をしている？

**q-bu.com のスマホ版を新規開発**している。

q-bu は子ども向けワークショップ用ツール。子どもが立方体（ブロック）を並べて3Dモデルを作り、
それを **3Dプリント** して実物を渡して喜んでもらう。

- **v1のゴール**：子どもが各自スマホで「ブロックを並べる → 送信 → 完了」。
  送信データはホストPCに集約 → STL変換 → 3D印刷 → 子どもに渡す。
- 詳細仕様 → [要件定義.md](要件定義.md)
- 将来拡張の方針 → [拡張性設計.md](拡張性設計.md)

5. **拡張性設計** … [拡張性設計.md](拡張性設計.md)。B2B拡張の土台（seam）を仕込み

## 6. 動かし方

**かんたん起動（Mac）**：`start.command` を**ダブルクリック** → サーバ起動＋先生用QR画面が自動で開く。

**手動起動**：
```bash
python3 host.py
```
起動すると先生用／子ども用のURLがターミナルに表示される。

- **先生用** `http://<MacのLAN IP>:8000/admin.html` … QRを画面表示（子どもが読む）
- **子ども用** `http://<MacのLAN IP>:8000/mock.html` … QRを読むとここが開く
- iPhoneは「ホーム画面に追加」で全画面アプリ風になる（`manifest.json`）。
- 送信データは `submissions/` に JSON で溜まる（1人1ファイル）。
- **ネット不要**：Three.js も QR ライブラリも `vendor/` にローカル同梱済み。
  会場のWi-Fiにインターネットが無くても動く。
