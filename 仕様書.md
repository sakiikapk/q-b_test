## 2. データスキーマ（`qbu.creation/1`）

保存先 `submissions/<id>_<YYYYMMDD-HHMMSS>.json`。**前方互換**が肝：受信側は中身を「そのまま」保存し、未知フィールドも壊さない。

```json
{
  "schema": "qbu.creation/1",
  "meta": {
    "appVersion": "0.1.0",
    "tenant": "default",        // B2B: 顧客単位
    "event":  "default",        // B2B: イベント単位
    "mode":   "model",          // 将来: paint / code ...
    "createdAt": "2026-06-17T12:01:01.327Z"   // UTC / ISO8601
  },
  "creator": { "id": "A-1376", "name": "" },
  "model": {
    "unit": 1,
    "bounds": [100, 100, 100],  // 編集可能空間（mock側 CONFIG.limits.bounds）
    "blocks": [                 // 立方体の集合（ボクセル）
      { "x": 0, "y": 0, "z": 0, "color": "#9AA0A6" }
    ]
  }
}
```

- `creator.id` … スマホ側でランダム発番（`A-XXXX`）。localStorage `qbu_id` に保持＝再接続でも維持。ファイル名にも使用。
- `model.blocks[].color` … 16進カラー。複数色対応の土台（現状パレットは1色）。
- バージョンを上げるときは `schema` を `qbu.creation/2` のように更新し、受信側の互換を保つ。

## 3. HTTP API（host.py）

| メソッド | パス | 用途 | 返り値 |
|---|---|---|---|
| GET | `/<静的ファイル>` | アプリ配信（mock/ar/admin/data/vendor 等） | ファイル |
| GET | `/lan` | LAN情報（QR生成用）。**Hostヘッダ対応** | `{ip, port, scheme, host, selfSigned, appUrl}` |
| GET | `/list` | 受信データ要約一覧（新しい順） | `{count, dir, items:[{file,id,name,mode,createdAt,blocks,colors,mtime}]}` |
| POST | `/submit` | 作品JSON保存（そのまま保存） | `{ok, saved}` |
| POST | `/ar?id=...` | iOS AR Quick Look 用 USDZ 一時保存 | `{ok, url}` |
| GET | `/key.pem`, `/.cert_ip` | 秘密鍵は配信拒否 | **403** |

- 全レスポンスに `Cache-Control: no-store`（常に最新を配信）。
- HTTPS：起動時に現在のLAN IPをSANに含む自己署名証明書を自動生成。**IPが変わると再発行**（`.cert_ip` で判定）。openssl 未導入時は http にフォールバック。
- `/lan` の `selfSigned` … LAN直アクセス（Host がIP）なら `true`、トンネル経由（Hostがドメイン）なら `false`。admin.html はこれで証明書導線の出し分け・QRの自動切替を行う。

## 4. フロント設計

### mock.html（子ども用アプリ）
- Three.js（importmap, `vendor/three/`）。タッチ操作：**面タップ=置く/消す**、**1本指ドラッグ=回す**、**ピンチ=ズーム**、視点ボタン。
- **CONFIG switchboard**（拡張のseam）：`branding / palette / features{...} / limits / tenant / event`。機能はフラグで出し分け（例：`features.ar` で「ARで見る」ボタンを表示）。
- 自動保存：作品を localStorage `qbu_blocks` に保存（ar.html と同一オリジンで共有・途中復元の土台）。
- 送信：`meshes` → `blocks[]` に変換して POST `/submit`。完了で受付番号（`A-XXXX`）を表示。

### ar.html（マーカーAR）
- A-Frame + AR.js（`vendor/arjs/`）。Hiroマーカー（`patt.hiro`／印刷は [marker.html](marker.html)）。
- 表示：localStorage `qbu_blocks` を読み、マーカー中央に**固定サイズ**（定数 `FIT_SCALE`）で配置。
- 実機UXの工夫（操作を安定させる制約）：
  - **面接着のみ**：既存ブロックの面をタップした時だけ、その面にくっつけて1個追加。床置き・浮き（つながらない島）は不可。
  - **最大サイズ 30×30×30マス**（定数 `LIMIT`）を超える置き方は禁止。
  - 起点キューブ：空なら原点に1個（全消ししても復活＝面が無くて詰むのを防ぐ）。
  - 見る向きは「マーカーを基準に iPhone を回り込む」運用（モデル回転UIは無し）。

## 5. AR / カメラ / HTTPS のハマりどころ（重要）

- **iOSのカメラ(getUserMedia)は「信頼されたHTTPS」が必須**。http では不可。自己署名はiPhoneに証明書をインストール＆**信頼ON**が必要。証明書はLAN IP固定で発行されるため、**会場のIPが変わると再インストール**が要る。
- **開発は cloudflared トンネルが楽**：`cloudflared tunnel --url https://localhost:8000 --no-tls-verify` で Google Trust Services 発行の本物の証明書付きURL（`https://*.trycloudflare.com`）が出る → **iPhoneは証明書インストール不要**。`テスト起動.command` が自動化。`/lan` が Host を見るので admin.html のQRも自動でトンネルURLになる。要インターネット。
- **AR.jsの黒画面**：AR.js はカメラ video を `z-index:-2` で置くため、黒背景の裏に回って真っ黒になる。ar.html のCSSで video/canvas を前面側へ上書き＋ `transform:translateZ(0)`（iOSのfixed video黒画面対策）。
- **映像と3Dのズレ/歪み**：カメラ映像(video)と3D(canvas)は**同じ拡縮**（両方 `object-fit:cover`）にしないと、立方体が直方体に歪む・位置がズレる。※2026-06-24時点で軽微な歪み/傾きが残課題（要詰め）。

## 6. 拡張のseam（土台）

- **CONFIG features**（mock.html）：`multiColor / textures / ar / posting / gallery …` をフラグで段階実装。
- **ボクセル→メッシュ共有パイプライン**：mock.html の cubesGroup（立方体集合）が、将来 **STL / GLB / AR / サムネ** の共通入力になる（mock.html 内コメントの ★seam 参照）。
- **前方互換JSON**：`schema` バージョン管理＋未知フィールド保持。
- 詳細・ロードマップ → [拡張性設計.md](拡張性設計.md)。

### 次の実装候補
1. **STL生成**（host側）：`submissions/*.json` の blocks → 三角形メッシュ → STL（プリント出力）。data.html の3Dプレビューが土台。
2. **複数色**：CONFIG.palette を増やし、blocks の color を活かす（mock/ar/プレビュー/将来GLB）。
3. **GLB書き出し → AR**：色付きメッシュを GLB 化し、iOS AR Quick Look（USDZ）/ WebXR へ。

## 7. セキュリティ / 運用メモ

- `.gitignore` 済み：`key.pem`（秘密鍵）・`cert.pem`・`.cert_ip`・`submissions/`・`*.log`・`ar_tmp/`。秘密鍵は配信も **403**。
- 受信データは「そのまま」保存＝前方互換を壊さない。
- ホストは **Python標準ライブラリのみ**（証明書生成に openssl を使うだけ）。会場のWi-Fiにインターネットが無くてもLAN版で動く。
- 想定ハード：ホスト＝Mac mini M4級、プリンタ＝単色×複数台並列（詳細は [要件定義.md](要件定義.md)）。
